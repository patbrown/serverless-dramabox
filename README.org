#+TITLE: Dramabox RunPod Serverless Worker

The world is a stage. Shakespeare.

* Purpose

This directory is a self-contained RunPod serverless worker for
=ResembleAI/Dramabox=.

It is meant to be deployed from GitHub through the RunPod browser UI.

* RunPod GitHub Deploy Settings

Use:

#+begin_src text
Repository root: resources/runpod/serverless/dramabox
Dockerfile path: Dockerfile
#+end_src

Recommended first endpoint:

#+begin_src text
Endpoint type: Queue
GPU: 48GB VRAM or larger
Container disk: 150GB
#+end_src

The request path is not a deploy setting. After RunPod creates the endpoint,
call =/runsync= through the RunPod API URL for that endpoint.

* Request

#+begin_src json
{
  "input": {
    "prompt": "A woman speaks warmly, \"Hello, how are you today?\"",
    "voice_ref": null,
    "cfg_scale": 2.5,
    "stg_scale": 1.5,
    "duration_multiplier": 1.1,
    "seed": 42,
    "watermark": true
  }
}
#+end_src

* Response

When =DRAMABOX_S3_BUCKET= is set, the worker uploads the WAV to S3 and returns
object metadata. Use =output_mode= =base64= only for smoke tests.

* Environment Variables

Set these in RunPod:

#+begin_src text
DRAMABOX_CACHE_DIR=/runpod-volume/dramabox-cache
DRAMABOX_OUTPUT_DIR=/tmp/dramabox-output
DRAMABOX_PRELOAD=1
DRAMABOX_COMPILE_MODEL=0
DRAMABOX_DTYPE=bf16
DRAMABOX_BNB_4BIT=1
DRAMABOX_S3_BUCKET=...
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
HF_TOKEN=...
#+end_src

Only the access key and secret key are sensitive. Bucket and region are runtime
configuration.

S3 behavior:

- If an S3 bucket is set and =DRAMABOX_OUTPUT_MODE= is absent, the default
  output mode is =s3=.
- =output_mode= may be =s3=, =base64=, or =both= per request.
- The default S3 prefix is =dramabox=.
- Presigned URLs default to 86400 seconds.
- Set =DRAMABOX_S3_PRESIGN_SECONDS= to =0= to skip presigned URLs.
- Optional =DRAMABOX_S3_ENDPOINT_URL= supports S3-compatible storage.
- Optional =DRAMABOX_S3_PUBLIC_BASE_URL= is only for public buckets with a
  stable base URL.
- Bucket env names accepted: =DRAMABOX_S3_BUCKET=, =S3_BUCKET=,
  =AWS_S3_BUCKET=, =AWS_S3_BUCKET_NAME=, =AWS_BUCKET_NAME=.

If =DRAMABOX_CACHE_DIR= is absent, the worker chooses:

1. =/runpod-volume/dramabox-cache= when =/runpod-volume= exists
2. =/workspace/dramabox-cache= when =/workspace= exists
3. =/tmp/dramabox-cache=

* Notes

- The worker clones Dramabox at image build time.
- The worker installs Dramabox requirements in the image.
- Model weights are downloaded at worker startup into the selected cache.
- =TTSServer= is loaded once per worker process and reused.
- =torch.compile= is disabled by default because the runtime image does not
  include a C compiler for Triton/Inductor JIT.
- Production output should use S3. Base64 remains available for smoke tests.
